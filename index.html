What Is JSON?

  1. JSON is a data interchange format
  2. JSON is programming language independent (JavaScript is not required to use it)
  3. JSON is based on the object literal notation of JavaScript (emphasis on the word “notation”)
  4. JSON represents data in a way that is friendly to universal programming con‐cepts

JSON Syntax

  1. a simple exmaple { "brand": "Crocs", "color": "pink", "size": 9, "hasLaces": false }
  2. JSON uses the colon character (:) to separate the names and values.The name is always on the left and the value is always on the right.
  3. JSON is based on JavaScript object literals, so it looks much the same. A JavaScript object literal does not need quotes around the name of the name-value pair. In JSON, it is absolutely required. In JavaScript, an object may have single quotes for syntax instead of double quotes.In JSON, only double quotes are used, and they are absolutely required around the name of the name-value pair.
  4. The MIME type for JSON is application/json.
  5. JSON files use the .json extension.

JSON Data Types

  1. The JSON data types are:
    • Object
    • String
    • Number
    • Boolean
    • Null
    • Array

  2. The JSON String Data Type

    • example: 

      { "animal" : "cat" }

    • A string value must always be surrounded in double quotes.

      This is not valid JSON
{
'title': 'This is my title.',
'body': 'This is the body.'
}

    • This code won’t work

      {
"promo": "Say "Bob's the best!" at checkout for free 8oz bag of kibble."
}

      Using a backslash character to escape quotes inside of strings fixes the problem

      {
 "promo": "Say \"Bob's the best!\" at checkout for free 8oz bag of kibble."
}

      In addition to the double quote and backslash characters, you must escape the follow‐ ing characters:

      • \/ (forward slash)
      • \b (backspace)
      • \f (form feed)
      • \t (tab)
      • \n (new line)
      • \r (carriage return)
      • \u followed by hexadecimal characters (e.g., the smiley emoticon \u263A)

  3. The JSON Number Data Type

    • example: 

      {
  "widgetInventory": 289,
  "sadSavingsAccount": 22.59,
  "seattleLatitude": 47.606209,
  "seattleLongitude": -122.332071,
  "earthsMass": 5.97219e+24
}

    • A number in JSON can be an integer, decimal, negative number, or an exponent.

  4. The JSON Boolean Data Type

    • example: 

      {
"toastWithBreakfast": false,
"breadWithLunch": true
}

  5. The JSON Null Data Type

    • In JSON, null must always be all lowercase characters.

  6. The JSON Array Data Type

    • example: 

      {
  "eggCarton": [
  "egg",
  null,
  "egg",
  "egg",
  "egg",
  "egg",
  "egg",
  "egg",
  "egg",
  "egg",
  5,
  "egg"
  ] }
    • The array is always surrounded in square brackets ([]).
    • In JSON, the mixing and matching of data types is valid.

  7. The JSON Object Data Type

    • The Object is always surrounded in square brackets ({}).
    • A JavaScript object can carry functions within, while a JSON object cannot carry any functions.
